# Version Control & You

Software updates could be for any reason.
Typically, fixing bugs or adding functionality.

Tracking these updates in real time is essential for:
- Software longevity
- Proper upkeep

When only a few files are involved or the files are not very large, this is very manageable. 
As that scale increases, individual tracking of updates becomes too time-consuming. 

These software updates (a.k.a. **Versions**) may not implement as intended as well.
Having files to revert to, helps prevent irreversible damage to a file.

In addition, this allows anyone (beit managers, developers, or end-users) to see progression.
With detailed changelogs, it can be significantly easier to demonstrate what has occurred.

Having a control center for this:
- Tracking Updates
- Filing Changelogs
- Archiving Files

Allows a content creator to worry less about administration and more about their software.
Version control may also be referred to as **Source** control, as it controls the source files.

**GitLab** or **GitHub** are control _centers_ while **Git** is the _control_.
